ในความเข้าใจของผมคิด mongodb คือ non rational database หรือ ก็คือ documentation data base โดยเก็บเป็นหน่วยข้อมูล เช่น collection ไป document ไป field โดยที่เราไม่ต้องสร้างการเชื่อมโยงเอง เปรียบเหมือน เรา สร้างโฟลเดอร์ในคอมที่ย่อยไปอีกแล้วก็ย่อยไปอีก ซึ่ีงมันง่ายกว่าและไม่ซับซ้อนเท่ากับ prostgresql แถมในส่วนข้อมูล mongodb แล้วสามารถ ทำการ aggregation pipe line ซึ่ง สามารถเก็บข้อมูลซ้อนข้อมูลได้ และส่งออกมากตาม ค่าที่เราตั้งไว้

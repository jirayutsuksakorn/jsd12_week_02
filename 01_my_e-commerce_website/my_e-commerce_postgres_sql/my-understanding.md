ผมคิดว่า postgresql ตามแบบที่ผมเข้าใจ คือ
1.เป็น data base แบบ rational ดดยเราต้องเป็นคนกำหนดความเยื่องโยงต่างๆขอ data ของเรา เช่น table column row คือ sql จะเก็บข้อมูลแบบเป็นระเบียบกว่า
2.ในส่วนของ data ที่เราสร้างขึ้นนั้น เราต้องเป้นคนกำหนด data type , primary key, foreigne key ซึ่งทำให้ มันเป็นระเบียบ และ ตรงกับเงื่อนไขข้อมูลได้ดีกว่า
3.การดึงข้อมูลจากหลายตารางมารวมกัน ในการ query ในการ query ครั้งเดียว ซึ่งต่างกับ mongodb ซึ่ง จะต้องใช้ aggregation ในการดึงข้อมูลมาสรุปแบบครบมากกว่า
